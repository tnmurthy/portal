# Project Overlord: FDE Handover Document
**Mission Control Agentic OS MVP**

## 🏗️ Architectural Core
Project Overlord is a **Forward Deployed Engineer (FDE)** grade orchestration platform. It is designed around the principle of **Deployment Parity**: one unified backbone with three distinct runtime fulfillment models (Shared, Dedicated, Sovereign).

### Key Components
1.  **The Lead Agent (Architect):** Generates tiered `SquadManifest` artifacts from vague mission briefs.
2.  **The Orchestrator (LangGraph):** A stateful, cyclic execution engine that manages agent debate and context hand-offs.
3.  **The State Manager (Postgres):** Single source of truth with real-time `pg_notify` triggers for low-latency observability.
4.  **The Resilience Mesh (Ollama Fallback):** Automatically reroutes missions to local hardware if cloud LLM quotas are hit.
5.  **The Dashboard (Next.js):** An FDE-grade console for real-time monitoring, HITL approval, and intelligence reporting.

## 🛠️ Operational Stack
*   **Backend:** FastAPI (Port 8001)
*   **Frontend:** Next.js 14 (Port 3001)
*   **Database:** PostgreSQL + `pgvector`
*   **Orchestration:** LangGraph + LangChain
*   **Tools:** Serper (Google Search), BeautifulSoup (Scraper)
*   **Sovereign Fallback:** Ollama (`qwen2.5-coder:7b`)

## 🚀 Getting Started
1.  **Launch Infra:** `cd sync-platform && docker-compose up -d`
2.  **Start Gateway:** `cd mission-control && python main.py`
3.  **Start Dashboard:** `cd mission-control/dashboard && npm run dev`
4.  **Configure `.env`:** Ensure `OPENAI_API_KEY`, `SERPER_API_KEY`, and `DATABASE_URL` are set in `mission-control/`.
